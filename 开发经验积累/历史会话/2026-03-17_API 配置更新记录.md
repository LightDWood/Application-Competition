## 五、API 调用流程

### 5.1 非流式调用
```
用户消息 
  ↓
构建消息数组（包含 system prompt + 历史对话 + 用户消息）
  ↓
调用 POST /chat/completions
  ↓
解析响应数据
  ↓
返回内容给用户
```

### 5.2 流式调用（SSE）
```
用户消息
  ↓
构建消息数组
  ↓
调用 POST /chat/completions (stream: true)
  ↓
获取 ReadableStream
  ↓
逐块读取数据（解析 SSE 格式：data: {...}\n\n）
  ↓
实时返回给前端
  ↓
接收到 [DONE] 标记，结束
```

## 六、错误处理

### 6.1 可能的错误类型

1. **网络错误**
   - API 服务不可达
   - 超时（60 秒）

2. **认证错误**
   - API Key 无效
   - API Key 过期

3. **请求错误**
   - 参数格式错误
   - 超出配额限制

4. **解析错误**
   - SSE 数据格式异常
   - JSON 解析失败

### 6.2 错误处理策略

```javascript
try {
  const response = await this.callChatAPI(messages, false)
  return {
    type: 'response',
    content: response.content,
    metadata: { model, usage }
  }
} catch (error) {
  console.error('调用 AI API 失败:', error)
  return {
    type: 'error',
    content: `抱歉，AI 服务暂时不可用：${error.message}`,
    metadata: { error: error.message }
  }
}
```
