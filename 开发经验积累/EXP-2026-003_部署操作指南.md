##  日常部署方式 (重要)

### 方式一：Git 一键部署 (推荐)

**适用场景**: 
-  日常功能更新
-  Bug 修复
-  代码迭代

**前提条件**:
-  代码已提交到 GitHub 仓库
-  服务器已配置 Git 和 SSH 密钥

**部署步骤**:

`ash
# 1. SSH 连接到服务器
ssh root@123.57.165.99
# 输入密码：H7$kB9@Sr#pQ2xR

# 2. 执行一键部署脚本
deploy-ai-contest

# 3. 验证部署
curl -I http://123.57.165.99/ai-contest/
curl http://123.57.165.99/ai-contest/api/health
`

**部署脚本说明**:

deploy-ai-contest 脚本会自动执行:
1. 从 GitHub 拉取最新代码
2. 安装/更新依赖 (
pm ci --production)
3. 重启 PM2 服务
4. 执行健康检查

### 方式二：手动 Git 部署

**适用场景**:
-  需要查看部署细节
-  部署脚本执行失败
-  需要回滚到特定版本

**部署步骤**:

`ash
# 1. SSH 连接到服务器
ssh root@123.57.165.99

# 2. 进入后端目录
cd /var/www/ai-contest/backend

# 3. 拉取最新代码
git pull origin main

# 4. 安装依赖
npm ci --production

# 5. 重启服务
pm2 restart ai-contest

# 6. 查看日志确认启动成功
pm2 logs ai-contest --lines 20

# 7. 健康检查
curl http://localhost:3001/health
`

---

### 方式三：本地构建 + SCP 上传

**适用场景**:
-  前端代码更新
-  Git 部署不可用
-  需要本地测试构建

**部署步骤**:

`powershell
# 1. 本地构建前端
cd ai-contest-web
npm run build

# 2. 上传前端文件到服务器
scp -r dist/* root@123.57.165.99:/var/www/ai-contest/frontend/

# 3. SSH 连接服务器重启服务
ssh root@123.57.165.99 "pm2 restart ai-contest"

# 4. 验证部署
curl -I http://123.57.165.99/ai-contest/
`

---

### 方式四：Rsync 增量部署

**适用场景**:
-  大文件更新
-  网络较慢
-  需要增量同步

**部署步骤**:

`ash
# 1. 本地构建
cd ai-contest-web
npm run build

# 2. 使用 Rsync 增量上传前端
rsync -avz --delete ./dist/ root@123.57.165.99:/var/www/ai-contest/frontend/

# 3. 重启服务
ssh root@123.57.165.99 "pm2 restart ai-contest"
`

##  运维命令速查

### PM2 命令

`ash
pm2 list          # 查看所有进程
pm2 logs ai-contest    # 查看日志
pm2 restart ai-contest # 重启应用
pm2 monit         # 查看监控
pm2 save          # 保存进程列表
`

### Nginx 命令

`ash

systemctl reload nginx # 重载配置
tail -f /var/log/nginx/error.log # 查看日志
