# PowerShell 命令格式错误经验总结

## 基本信息
- 经验编号：EXP-2026-002
- 问题类型：PowerShell 命令/语法错误
- 发生时间：2026-03-17

## 常见错误类型

### 错误 1：Markdown 符号被当作 PowerShell 命令

错误表现：
- 列表项、标题、分隔线等 Markdown 符号直接执行
- 例如：- 列表项、### 标题、---、**粗体**

错误信息：
- 一元运算符"-"后面缺少表达式
- 无法识别为 cmdlet、函数、脚本文件

根本原因：
- PowerShell 将"-"识别为一元运算符或参数前缀
- "#"是注释，但"###"后面的内容仍会被解析
- "---"被解析为减法运算

正确做法：
必须使用 Here-String 包裹多行文本
$content = @""内容"@
$content | Out-File -FilePath "文件.md" -Encoding utf8

### 错误 2：逗号分隔多个命令

错误表现：
ls .\需求\, ls .\开发经验积累\

错误信息：
- 未能找到路径
- 找不到路径

根本原因：
- PowerShell 中逗号用于创建数组，不是命令分隔符

正确做法：
- 使用分号：ls .\需求\; ls .\开发经验积累\
- 分行执行：
  ls .\需求\
  ls .\开发经验积累\

### 错误 3：Here-String 语法使用不当

错误表现：
- 边界符不匹配
- 边界符前后有空格
- 内容中包含边界符

正确做法：
- 开始符后必须立即换行
- 结束符必须顶格（第一列）
- 双引号版本支持变量展开

### 错误 4：路径包含特殊字符未转义

错误表现：
dir "路径" /a

错误信息：
- 第二个路径段不得为驱动器或 UNC 名称

根本原因：
- /a 是 CMD 语法，不是 PowerShell 语法

正确做法：
Get-ChildItem "路径" -Force

## 最佳实践

1. 多行文本处理
   - 使用 Here-String
   - 指定 UTF8 编码
   - 管道输出到 Out-File

2. 命令分隔
   - 使用分号或分行
   - 不使用逗号

3. Here-String 规则
   - 开始符后立即换行
   - 结束符必须顶格
   - 双引号版本支持变量

4. 路径处理
   - 使用 PowerShell 原生语法
   - 避免 CMD 语法

## 关键要点

PowerShell 语法特点：
1. "-"是一元运算符或参数前缀
2. "#"是注释
3. "---"被解析为减法运算
4. 命令分隔使用分号或换行

Here-String 规则：
1. 开始符后必须立即换行
2. 结束符必须顶格
3. 双引号版本支持变量展开

Markdown 内容处理：
1. 必须放在字符串或 Here-String 中
2. 使用-Encoding utf8 避免中文乱码
3. 通过管道传递给 Out-File
