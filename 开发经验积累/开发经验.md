## 一、文件操作最佳实践

### 1.1 文件创建方式选择

**问题**: Write 工具可能报告成功但实际未创建文件

**解决方案**: 优先使用 PowerShell 命令行

```powershell
# 推荐方式：使用 Here-String + Out-File
$content = @"
# 文档标题
内容...
"@
$content | Out-File -FilePath ".\需求\文件名.md" -Encoding utf8

# 验证文件创建
Test-Path ".\需求\文件名.md"
```

**关键要点**:
- 必须指定 UTF8 编码避免中文乱码
- 创建后必须验证文件是否存在
- 优先使用相对路径而非绝对路径

### 1.2 路径处理规范

**问题**: 路径权限白名单限制导致操作失败

**错误示例**:
```
Out-File : operation not permitted
path not in allowlist
```

**解决方案**:
```powershell
# 错误：使用绝对路径
Out-File -FilePath "d:\DFX\Code\AIbisai-wangzhan\需求\文件.md" -Encoding UTF8

# 正确：使用相对路径
cd 'd:\DFX\Code\AIbisai-wangzhan'
Out-File -FilePath ".\需求\文件.md" -Encoding UTF8

# 或使用重定向符号
'' > '需求\文件.md'
```

**最佳实践**:
1. 始终使用相对路径
2. 避免在 `.trae`、`.vscode`、`.git` 目录操作文件
3. 确保目标目录存在后再创建文件

## 二、PowerShell 命令规范

### 2.1 多行文本处理

**错误**: Markdown 符号被当作 PowerShell 命令

```powershell
# 错误：直接执行 Markdown 内容
- 列表项
### 标题
**粗体**

# 正确：使用 Here-String 包裹
$content = @"
- 列表项
### 标题
**粗体**
"@
$content | Out-File -FilePath "file.md" -Encoding utf8
```

### 2.2 命令分隔

**错误**: 使用逗号分隔多个命令

```powershell
# 错误
ls .\需求\, ls .\开发经验积累\

# 正确：使用分号
ls .\需求\; ls .\开发经验积累\

# 或分行执行
ls .\需求\
ls .\开发经验积累\
```

### 2.3 Here-String 语法规则

```powershell
# 正确用法
$content = @"
第一行内容
第二行内容
变量展开: $variable
"@  # 结束符必须顶格

# 单引号版本（不展开变量）
$content = @'
第一行内容
第二行内容
'@
```

**关键规则**:
- 开始符 `@"` 后必须立即换行
- 结束符 `"@` 必须顶格（第一列）
- 双引号版本支持变量展开
- 单引号版本不展开变量

### 2.4 路径命令语法

```powershell
# 错误：使用 CMD 语法
dir "路径" /a

# 正确：使用 PowerShell 语法
Get-ChildItem "路径" -Force
```

## 三、部署操作指南

### 3.1 服务器信息

| 项目 | 值 |
|------|-----|
| 服务器 IP | 123.57.165.99 |
| SSH 用户 | root |
| 项目路径 | /var/www/ai-contest/ |
| 应用端口 | 3001 |

### 3.2 部署方式优先级

#### 方式一：Git 一键部署（推荐）

```bash
# SSH 连接服务器
ssh root@123.57.165.99

# 执行一键部署脚本
deploy-ai-contest

# 验证部署
curl -I http://123.57.165.99/ai-contest/
curl http://123.57.165.99/ai-contest/api/health
```

#### 方式二：手动 Git 部署

```bash
ssh root@123.57.165.99
cd /var/www/ai-contest/backend
git pull origin main
npm ci --production
pm2 restart ai-contest
pm2 logs ai-contest --lines 20
```

#### 方式三：本地构建 + SCP 上传

```powershell
# 本地构建
cd ai-contest-web
npm run build

# 上传到服务器
scp -r dist/* root@123.57.165.99:/var/www/ai-contest/frontend/

# 重启服务
ssh root@123.57.165.99 "pm2 restart ai-contest"
```

### 3.3 运维命令速查

```bash
# PM2 命令
pm2 list                  # 查看所有进程
pm2 logs ai-contest       # 查看日志
pm2 restart ai-contest    # 重启应用
pm2 monit                 # 查看监控

# Nginx 命令
nginx -t                  # 测试配置
systemctl reload nginx    # 重载配置

# Git 命令
git pull origin main      # 拉取代码
git log --oneline -10     # 查看日志
git reset --hard <commit> # 回滚版本
```

## 四、故障排查清单

### 4.1 文件操作故障

| 问题 | 排查步骤 | 解决方案 |
|------|---------|---------|
| 文件创建失败 | 1. 检查路径格式<br>2. 验证目录存在<br>3. 检查权限 | 使用相对路径 + PowerShell 命令 |
| 中文乱码 | 检查编码设置 | 指定 `-Encoding utf8` |
| 权限拒绝 | 检查路径白名单 | 使用相对路径，避免受保护目录 |

### 4.2 部署故障

| 问题 | 排查步骤 | 解决方案 |
|------|---------|---------|
| 前端无法访问 | 1. 检查文件存在<br>2. 测试 Nginx 配置<br>3. 查看服务状态 | `ls -la /var/www/ai-contest/frontend/` |
| 后端服务异常 | 1. 查看错误日志<br>2. 健康检查 | `pm2 logs ai-contest --err` |
| Git 拉取失败 | 1. 测试 SSH 连接<br>2. 检查远程仓库 | `ssh -T git@github.com` |

## 五、核心经验总结

### 5.1 工具使用原则

1. **不要盲目相信工具返回** - 必须进行二次验证
2. **优先使用底层命令** - PowerShell 命令优于高级工具
3. **路径问题要谨慎** - 优先使用相对路径

### 5.2 开发规范

1. **文件创建**: PowerShell 命令 > Write 工具
2. **路径使用**: 相对路径 > 绝对路径
3. **编码指定**: 必须指定 UTF8 编码
4. **命令分隔**: 分号或换行，不用逗号

### 5.3 部署规范

1. **部署前检查**: 代码测试通过、已提交 Git、本地构建成功
2. **部署后验证**: 前端可访问、API 响应正常、日志无异常
3. **部署方式**: Git 一键部署 > 手动部署 > SCP 上传
