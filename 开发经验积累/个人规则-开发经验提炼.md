## 规则一：PowerShell 命令规范

### 规则内容

```
在 Windows PowerShell 环境下执行命令时，必须遵循以下规范：

1. 多行文本处理：
   - 必须使用 Here-String 包裹 Markdown 等特殊格式内容
   - 开始符 @" 后必须立即换行
   - 结束符 "@ 必须顶格（第一列）
   - 双引号版本支持变量展开，单引号版本不展开

2. 命令分隔：
   - 使用分号（;）或换行分隔多个命令
   - 禁止使用逗号分隔命令

3. 路径命令：
   - 使用 PowerShell 原生语法（Get-ChildItem、Test-Path 等）
   - 避免使用 CMD 语法（dir /a 等）

4. 文件编码：
   - 创建或写入文件时必须指定 UTF8 编码
   - 使用 -Encoding utf8 参数
```

### 适用场景

- 所有 Windows 环境下的开发工作
- 跨项目通用

## 规则二：文件操作最佳实践

### 规则内容

```
执行文件操作时，必须遵循以下规范：

1. 路径使用：
   - 优先使用相对路径，避免绝对路径
   - 避免在 .trae、.vscode、.git 等系统目录操作文件
   - 确保目标目录存在后再创建文件

2. 文件创建验证：
   - 关键文件（配置文件、部署文件）：必须验证
   - 普通文件：建议验证
   - 验证方式：使用 Test-Path 或 Read 工具确认

3. 编码处理：
   - 所有文件操作必须指定 UTF8 编码
   - 避免中文乱码问题

4. Write 工具使用注意：
   - Write 工具对包含中文或空格的绝对路径可能解析失败
   - 遇到文件创建问题时，优先使用相对路径
   - 或使用 PowerShell Out-File 命令作为替代方案
```

### 适用场景

- 所有文件操作场景
- 跨项目通用

## 规则三：工具使用原则

### 规则内容

```
使用 Trae 开发工具时，必须遵循以下原则：

1. 工具使用优先级：
   - 优先使用 Trae 提供的工具（Write、Read、Glob、Grep 等）
   - 这些工具经过优化，支持跨平台和错误处理
   - 仅在工具无法满足需求时才使用底层命令

2. 问题诊断流程：
   - 当工具返回异常时，使用 PowerShell 命令进行诊断
   - 对于已知问题场景（如中文路径、特殊字符），可主动使用 PowerShell
   - 记录问题原因和解决方案，形成经验积累

3. 路径处理：
   - 优先使用相对路径
   - 注意中文路径和空格的处理
   - 遇到路径问题时，尝试切换相对/绝对路径

4. 错误处理：
   - 关键操作（部署、配置修改）建议二次验证
   - 使用 Read 工具或 Test-Path 验证文件操作结果
   - 记录错误信息和解决过程
```

### 适用场景

- 所有开发场景
- 跨项目通用

## 规则四：文档编写规范

### 规则内容

```
编写开发文档时，必须遵循以下规范：

1. 文档结构：
   - 必须包含：问题描述、解决方案、最佳实践
   - 可选包含：问题诊断过程、根本原因分析

2. 代码示例：
   - 必须提供正确示例和错误示例对比
   - 代码块必须指定语言标签

3. 表格使用：
   - 使用表格对比不同方案的优劣
   - 使用表格整理故障排查清单

4. 版本信息：
   - 记录文档版本、创建时间、更新时间
   - 标注文档状态（草稿/已验证/已过时）
```

### 适用场景

- 编写技术文档
- 记录开发经验

## 快速配置指南

### 在 TRAE 中配置个人规则

1. 打开设置中心
   - IDE 模式：右上角设置图标
   - SOLO 模式：对话面板右上角设置图标

2. 进入规则和技能面板
   - 左侧导航栏选择「规则和技能」

3. 创建个人规则
   - 在「个人规则」部分点击「+ 创建」
   - 输入规则内容
   - 点击「保存」

### 推荐配置顺序

| 顺序 | 规则名称 | 优先级 | 说明 |
|------|---------|--------|------|
| 1 | PowerShell 命令规范 | 高 | 避免语法错误 |
| 2 | 文件操作最佳实践 | 高 | 避免文件操作失败 |
| 3 | 工具使用原则 | 中 | 提升开发效率 |
| 4 | 文档编写规范 | 低 | 规范文档质量 |
