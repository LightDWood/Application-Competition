# 需求收敛模块 - 开发需求文档

## 1. 项目概述

### 1.1 项目背景
基于已有的 requirement-convergence-skill 技能（能够负责收敛用户需求，形成详细的需求规格说明书），构建一套完整的多用户程序框架，支持用户通过 Web 界面使用该技能进行需求收敛工作。

### 1.2 项目目标
- 构建多用户体系，实现用户数据隔离
- 提供友好的 Web 界面，支持用户与技能进行交互
- 实现会话管理和历史版本管理
- 确保系统高可用性和并发处理能力

## 2. 系统架构

### 2.1 整体架构

```

                      前端层                              
       
   历史会话     会话问答窗口   文档下载及版本管理      
       

                          

                      API 网关层                           
  - 负载均衡  - 并发控制  - 会话管理  - 认证授权          

                          

                      后端服务层                          
       
   用户服务     会话服务       文档存储服务           
       

                          

                      Skill 服务层                         
            requirement-convergence-skill（需求收敛技能）                  

                          

                      数据存储层                          
       
   用户数据库   会话数据库     文件存储系统           
       

```

### 2.2 技术栈建议
- **前端**: React/Vue/Angular + TypeScript
- **后端**: Node.js/Python/Go（根据现有项目技术栈选择）
- **数据库**: PostgreSQL/MySQL（用户数据、会话元数据）
- **文件存储**: MinIO/S3/本地文件系统
- **缓存**: Redis（会话缓存、并发控制）
- **消息队列**: RabbitMQ/Kafka（异步任务处理）

## 5. 数据模型设计

### 5.1 用户表 (users)
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    avatar_url VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login_at TIMESTAMP,
    is_active BOOLEAN DEFAULT TRUE
);
```

### 5.2 会话表 (sessions)
```sql
CREATE TABLE sessions (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    title VARCHAR(200) NOT NULL,
    status VARCHAR(20) DEFAULT 'active', -- active, archived, deleted
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_message_at TIMESTAMP,
    metadata JSONB
);
```

### 5.3 消息表 (messages)
```sql
CREATE TABLE messages (
    id UUID PRIMARY KEY,
    session_id UUID REFERENCES sessions(id) ON DELETE CASCADE,
    role VARCHAR(20) NOT NULL, -- user, assistant, system
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    metadata JSONB
);
```

### 5.4 文档表 (documents)
```sql
CREATE TABLE documents (
    id UUID PRIMARY KEY,
    session_id UUID REFERENCES sessions(id) ON DELETE CASCADE,
    title VARCHAR(200) NOT NULL,
    file_path VARCHAR(500) NOT NULL,
    file_size BIGINT,
    file_type VARCHAR(50),
    current_version INTEGER DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    metadata JSONB
);
```

### 5.5 文档版本表 (document_versions)
```sql
CREATE TABLE document_versions (
    id UUID PRIMARY KEY,
    document_id UUID REFERENCES documents(id) ON DELETE CASCADE,
    version_number INTEGER NOT NULL,
    content TEXT NOT NULL,
    file_path VARCHAR(500),
    change_summary VARCHAR(500),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    created_by UUID REFERENCES users(id),
    metadata JSONB,
    UNIQUE(document_id, version_number)
);
```

## 6. 核心流程

### 6.1 用户会话流程
```
用户登录  创建/选择会话  输入需求  
Agent 处理（调用 requirement-convergence-skill）  生成响应/文档  
存储会话记录  文档版本管理  用户下载/查看
```

### 6.2 Skill 调用流程
```
接收用户消息  队列管理  调用 requirement-convergence-skill  
流式返回结果  解析响应  存储对话记录  
检测文档生成  存储文档及版本  返回前端
```

### 6.3 文档版本管理流程
```
Agent 生成文档  检测文档变更  
创建新版本记录  存储新版本内容  
更新文档元数据  通知前端更新
```
