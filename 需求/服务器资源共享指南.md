## 四、Web 服务配置 (Nginx)

### 4.1 当前配置

Nginx 配置文件位置：`/etc/nginx/nginx.conf` 或 `/etc/nginx/conf.d/`

当前抽奖系统配置：

```nginx
upstream lottery_backend {
    server 127.0.0.1:3000;
    keepalive 64;
}

server {
    listen 80;
    server_name _;
    client_max_body_size 10M;
    
    location / {
        root /var/www/lottery/frontend;
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://lottery_backend;
    }
    
    location /prizes {
        proxy_pass http://lottery_backend;
        expires 30d;
    }
}
```

### 4.2 添加新项目配置

**方案一：使用不同端口**

```nginx
# 新项目后端
upstream your_project_backend {
    server 127.0.0.1:3001;  # 使用不同端口
    keepalive 64;
}

# 新项目前端
location /your-project {
    alias /var/www/your-project/frontend;
    try_files $uri $uri/ /your-project/index.html;
}

# 新项目 API
location /your-project/api {
    rewrite ^/your-project/api/(.*) /api/$1 break;
    proxy_pass http://your_project_backend;
}
```

**方案二：使用子域名（需要配置 DNS）**

```nginx
server {
    listen 80;
    server_name your-project.example.com;
    
    location / {
        root /var/www/your-project/frontend;
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://your_project_backend;
    }
}
```

### 4.3 重载 Nginx 配置

```bash
# 测试配置
nginx -t

# 重载配置
nginx -s reload
```

## 七、认证机制参考

### 7.1 JWT 认证

当前抽奖系统使用 JWT 进行用户认证：

```javascript
const jwt = require('jsonwebtoken');

const JWT_SECRET = 'your_new_secret_key';  // 新项目请使用不同的密钥
const JWT_EXPIRES_IN = '24h';

// 生成 Token
const token = jwt.sign({ userId: 123 }, JWT_SECRET, { expiresIn: JWT_EXPIRES_IN });

// 验证 Token
const decoded = jwt.verify(token, JWT_SECRET);
```

### 7.2 管理员认证

当前系统使用 `x-admin-key` 请求头验证管理员身份：

```javascript
const ADMIN_KEY = 'your_new_admin_key';  // 新项目请使用不同的密钥

// 中间件验证
if (ctx.headers['x-admin-key'] === ADMIN_KEY) {
    // 验证通过
}
```

## 十一、安全建议

1. **数据库用户隔离**：每个项目使用独立的数据库用户，避免共用 root
2. **密钥隔离**：每个项目使用不同的 JWT_SECRET 和 ADMIN_KEY
3. **Redis 键前缀**：使用不同的键前缀避免数据冲突
4. **定期备份**：配置数据库定时备份
5. **日志监控**：使用 PM2 监控应用状态
