# 用户注册模块测试计划

## 一、测试目标

测试用户注册功能的完整性和健壮性，确保：
1. 用户可以成功注册新账号
2. 数据验证逻辑正确（唯一性、格式等）
3. JWT Token 正确生成和返回
4. 错误处理机制完善

## 二、测试环境

- **后端服务**: http://localhost:3000
- **注册接口**: POST /api/user/register
- **数据库**: MySQL (ai_contest)

## 三、测试用例

### 3.1 正常场景测试

#### 用例 1: 成功注册新用户
**请求**:
```bash
POST http://localhost:3000/api/user/register
Content-Type: application/json

{
  "username": "testuser001",
  "email": "testuser001@example.com",
  "password": "Test123456"
}
```

**预期响应**:
```json
{
  "code": 0,
  "message": "注册成功",
  "data": {
    "token": "<JWT_TOKEN>",
    "user": {
      "id": "<UUID>",
      "username": "testuser001",
      "email": "testuser001@example.com",
      "avatar_url": null,
      "created_at": "<TIMESTAMP>"
    }
  }
}
```

**验证点**:
- [ ] 响应状态码 201
- [ ] code 字段为 0
- [ ] message 为"注册成功"
- [ ] token 不为空且格式正确（JWT 三段式）
- [ ] user 对象包含所有必要字段
- [ ] 数据库中成功创建用户记录
- [ ] 密码已加密存储（bcrypt hash）

## 四、测试步骤

### 步骤 1: 启动后端服务
```bash
cd ai-contest-web/server
node app.js
```

### 步骤 2: 验证服务健康状态
```bash
Invoke-WebRequest -Uri http://localhost:3000/health -UseBasicParsing
```

### 步骤 3: 执行正常注册测试
```bash
$body = @{
    username = "testuser001"
    email = "testuser001@example.com"
    password = "Test123456"
} | ConvertTo-Json

Invoke-WebRequest -Uri "http://localhost:3000/api/user/register" `
    -Method POST `
    -ContentType "application/json" `
    -Body $body `
    -UseBasicParsing
```

### 步骤 4: 执行异常场景测试
依次测试用例 2-6 的异常场景

### 步骤 5: 验证数据库记录
```sql
SELECT * FROM users WHERE username = 'testuser001';
```

### 步骤 6: 验证 Token 有效性
使用返回的 Token 访问需要认证的接口：
```bash
GET http://localhost:3000/api/user/profile
Authorization: Bearer <TOKEN>
```

## 七、风险评估

1. **数据污染风险**: 测试数据可能影响开发环境
   - 应对：使用测试专用账号，测试后清理数据

2. **服务稳定性风险**: 频繁测试可能影响服务
   - 应对：控制测试频率，避免并发请求

3. **安全风险**: 测试账号密码不应提交到代码库
   - 应对：使用环境变量或临时配置
