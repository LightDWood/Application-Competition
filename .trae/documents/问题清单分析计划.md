## 详细分析与验证结果

### 问题1 & 问题2：新建会话 401 错误 + 会话无名字

**错误信息**:
```
PUT http://localhost:3000/api/sessions/9fe3c3d7-0e3e-46ff-8a22-8384fc363b6e 401 (Unauthorized)
更新标题失败: {code: 401, message: '未提供认证令牌'}
```

#### 代码调用链追踪

1. **前端 `ChatWindow.vue:228`**:
   ```javascript
   const saveTitle = async () => {
     const res = await sessionApi.update(sessionId, { title: editTitle.value })
   }
   ```

2. **API 调用分析**:

   **路径A - 使用 `sessionApi.update()` (非v2)**:
   - 调用: `api.put('/sessions/${id}', data)` → `PUT /api/sessions/{id}`
   - 使用的Token: `localStorage.getItem('admin_token')` (来自 `api/index.js` 拦截器)
   - 后端路由: `session.js:116` → `router.put('/:id', userAuthMiddleware, ...)`
   - **结果**: 需要 `admin_token`，但用户只有 `user_token`，会返回 401

   **路径B - 使用 `sessionApi.v2.update()` (v2版本)**:
   - 调用: `api.put('/v2/sessions/${id}', data)`
   - 使用的Token: `localStorage.getItem('admin_token')` (来自 `api/index.js` 拦截器)
   - 后端路由: `v2.js:699` → `router.put('/sessions/:id', async ...)` **无认证中间件**
   - **结果**: 不会返回401，但会失败

3. **v2.js 路由无认证验证**:
   ```javascript
   // v2.js:699 - 没有 userAuthMiddleware
   router.put('/sessions/:id', async (req, res) => {
   ```

#### ⚠️ 验证结论

**问题1 (401错误)**:
- ✅ **代码层面确认**: `sessionApi.update()` 使用 `admin_token`，但后端 `session.js:116` 需要 `user_token`
- ⚠️ **实际情况**: 用户报告使用 `/api/sessions/{id}` 返回401，说明代码路径确实是问题1描述的
- ⚠️ **待确认**: 需要实际运行环境验证

**问题2 (会话无名字)**:
- ✅ **代码层面确认**: 如果 `saveTitle` 失败，会话标题不会更新
- ⚠️ **待确认**: 需要实际运行环境验证

#### 解决方案

**修改 `ai-contest-web/src/api/session.js`**:

```javascript
// 修改 update 方法，为非v2的update添加user_token
update(id, data) {
  const token = localStorage.getItem('user_token')
  return api.put(`/sessions/${id}`, data, {
    headers: {
      Authorization: `Bearer ${token}`
    }
  })
}

// v2.update 保持不变，因为它调用的后端v2路由没有认证
v2: {
  update(id, data) {
    return api.put(`${V2_BASE}/sessions/${id}`, data)
  },
}
```

### 问题3：提示词优化 - 业务需求收敛完毕后自动生成需求收敛文档

**期望行为**: 当业务需求已收敛完毕后，系统应自动生成需求收敛文档，并在会话中提供下载链接。

#### 当前实现确认

1. **归档功能已实现** (`v2.js:473-519`):
   - 触发条件: `content.includes('归档')` - 用户必须手动输入"归档"
   - 文档生成: `generateRequirementDoc()` 生成固定模板的MD文档

   - ❌ 没有在检测到收敛完成后自动提示用户归档

#### ✅ 验证结论

### 问题4：归档时没有回复内容，且无下载链接

**错误现象**: 用户输入"归档"后，只看到空白回复，没有看到归档成功的内容和下载链接。

#### 代码追踪

1. **归档逻辑 `v2.js:473-519`**:
   ```javascript
   if (content.includes('归档')) {
     // ...生成文档...
     const artifactId = await artifactService.saveArtifact(id, {
       type: 'requirement-doc',
       name: `需求文档_${Date.now()}`,
       content: docContent
     })

     const artifactData = artifactService.getArtifact(artifactId)
     // ❌ artifactData 将为 null，因为 .md 文件不是 .json 文件

     const downloadPath = artifactData
       ? `/artifacts/${id}/${artifactData.fileName}`
       : `/artifacts/${id}/需求文档_${Date.now()}.md`  // fallback路径

     const archiveResponseContent = `✅ 归档成功！\n📄 文档已保存\n📥 下载链接: ${downloadPath}\n`

     res.write(`data: ${JSON.stringify({ type: 'chunk', content: archiveResponseContent })}\n\n`)
     // ...保存消息...
   }
   ```

2. **ArtifactService 路径不匹配**:
   - `saveArtifact()` 保存 `.md` 文件: `需求文档_{timestamp}.md`
   - `getArtifact()` 只查找 `.json` 文件: `${artifactId}.json`

   ```javascript
   // ArtifactService.js:89-100
   for (const [sessionId] of this.artifacts) {
     const artifactPath = path.join(this.baseDir, sessionId, `${artifactId}.json`);
     // ❌ 只查找 .json 文件，不查找 .md 文件
   }
   ```

3. **`getRequirementDoc()` 正确但未被使用**:
   ```javascript
   // ArtifactService.js:118-143
   getRequirementDoc(sessionId) {
     // 这个方法正确查找 .md 文件
     if (file.startsWith('需求文档_') && file.endsWith('.md')) {
       // ...
     }
   }
   ```

#### ✅ 验证结论

**问题4属实**: `getArtifact()` 无法找到 `.md` 文件，导致 `downloadPath` 使用了不正确的 fallback 路径，且归档消息可能正确发送但下载链接无效。

#### 解决方案

**修改 `ai-contest-web/server/services/core/ArtifactService.js`**:

```javascript
getArtifact(artifactId) {
  // 先检查内存缓存
  if (this.artifacts.has(artifactId)) {
    return this.artifacts.get(artifactId);
  }

  // 尝试查找 .json 文件（现有逻辑）
  // ...现有代码...

  // 新增：尝试在session目录中查找 .md 文件
  for (const sessionDir of fs.readdirSync(this.baseDir)) {
    const sessionPath = path.join(this.baseDir, sessionDir);
    if (!fs.statSync(sessionPath).isDirectory()) continue;

    try {
      const files = fs.readdirSync(sessionPath);
      for (const file of files) {
        // 匹配需求文档（通过 artifactId 或文件名模式）
        if (file.endsWith('.md') && (file.includes(artifactId) || file.startsWith('需求文档_'))) {
          const filePath = path.join(sessionPath, file);
          return {
            id: artifactId,
            fileName: file,
            filePath: filePath,
            type: 'requirement-doc'
          };
        }
      }
    } catch (e) {
      continue;
    }
  }

  return null;
}
```
