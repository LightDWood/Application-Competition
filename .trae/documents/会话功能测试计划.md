## 三、测试用例设计

### 3.1 核心测试：AI API 集成验证 (最高优先级)

#### 用例 AI-01: 验证 AI API 调用成功
**测试目标**: 确认系统能够正确调用 requirement-convergence-skill

**前置条件**: 
- 已创建会话
- 已获取有效 Token

**测试步骤**:
1. 发送包含需求触发词的消息 ("我有个想法")
2. 检查响应内容是否包含 AI 的启发式提问
3. 验证响应格式和结构

**请求**:
```json
POST /api/sessions/{sessionId}/messages/chat
Content-Type: application/json
Authorization: Bearer {token}

{
  "content": "我有个想法，想做一个任务管理系统"
}
```

**预期响应** (SSE 流式):
```
data: {"type":"chunk","content":"感谢您","done":false}

data: {"type":"chunk","content":"提出需求","done":false}

...

data: {"type":"complete","content":"","metadata":{"stage":"initial"},"done":true}
```

**验证点**:
- [ ] SSE 连接成功建立 (Content-Type: text/event-stream)
- [ ] 响应包含多个 chunk 数据块
- [ ] 最终返回 complete 标记
- [ ] 响应内容包含启发式提问
- [ ] metadata 包含 stage 信息
- [ ] 消息已保存到数据库

**预期内容特征**:
```
感谢您提出需求！我来帮您梳理和收敛需求。

**为了更好地帮助您，我想先了解一些关键信息：**

## 🎯 核心问题

1. **用户是谁？** 这个功能主要面向哪些用户群体？
2. **解决什么问题？** 用户目前是如何解决这个问题的？核心痛点是什么？
3. **预期目标？** 您希望达成什么样的效果？如何衡量成功？
```

#### 用例 AI-02: 验证需求收敛流程
**测试目标**: 验证 AI 能够根据对话进度调整响应

**测试步骤**:
1. 第一轮：发送初始需求 ("我想做个请假管理系统")
2. 第二轮：补充信息 ("主要是给 HR 部门使用的")
3. 第三轮：确认完成 ("差不多了")

**预期响应**:
- 第一轮：启发式提问 (type: clarification)
- 第二轮：进一步澄清 (type: clarification)  
- 第三轮：生成需求文档 (type: summary)

**验证点**:
- [ ] 不同阶段返回不同的响应类型
- [ ] 第三轮生成需求规格说明书
- [ ] metadata.stage 正确变化 (initial → clarifying → baseline)

#### 用例 AI-03: 验证触发词识别

## 四、测试步骤

### 步骤 1: 准备工作
```powershell
# 1. 启动后端服务
cd ai-contest-web\server
node app.js

# 2. 验证服务健康
Invoke-WebRequest -Uri http://localhost:3000/health -UseBasicParsing

# 3. 登录获取 Token
$loginBody = @{
    username = "test"
    password = "123456"
} | ConvertTo-Json

$loginResponse = Invoke-WebRequest -Uri "http://localhost:3000/api/user/login" `
    -Method POST `
    -ContentType "application/json" `
    -Body $loginBody `
    -UseBasicParsing

$token = ($loginResponse.Content | ConvertFrom-Json).data.token
Write-Host "Token: $token"
```

### 步骤 2: 创建测试会话
```powershell
$sessionBody = @{
    title = "AI 功能测试会话"
} | ConvertTo-Json

$sessionResponse = Invoke-WebRequest -Uri "http://localhost:3000/api/sessions" `
    -Method POST `
    -ContentType "application/json" `
    -Body $sessionBody `
    -Headers @{ Authorization = "Bearer $token" } `
    -UseBasicParsing

$sessionId = ($sessionResponse.Content | ConvertFrom-Json).data.id
Write-Host "Session ID: $sessionId"
```

### 步骤 3: 测试 AI API 调用
```powershell
# 使用 curl 测试 SSE (PowerShell 不支持 SSE)
curl -N -X POST "http://localhost:3000/api/sessions/$sessionId/messages/chat" `
  -H "Authorization: Bearer $token" `
  -H "Content-Type: application/json" `
  -d '{"content":"我有个想法，想做一个任务管理系统"}'
```

### 步骤 4: 验证数据库记录
```sql

-- 查看消息记录
```

## 五、AI API 集成验证详细方案

### 5.1 验证策略

由于 skillService 当前是**模拟实现**(未真正调用外部 API),测试重点为:

1. **服务层逻辑验证**: 验证 prompt 构建和响应生成逻辑
2. **响应格式验证**: 验证 SSE 流式响应格式正确
3. **对话流程验证**: 验证不同阶段的响应类型正确
4. **集成点验证**: 验证 routes → service 的调用链路

### 5.2 关键代码路径

```
routes/session.js (POST /:sessionId/messages/chat)
    ↓
routes/message.js (处理消息存储)
    ↓
services/skillService.js (processMessage / streamResponse)
    ↓
services/skillService.js (callSkill - 模拟 AI 响应)
```

### 5.3 验证点清单

#### 后端验证
- [ ] skillService.processMessage() 被调用
- [ ] buildPrompt() 正确构建 prompt
- [ ] callSkill() 根据关键词返回不同类型响应
- [ ] streamResponse() 产生流式输出
- [ ] SSE 头部设置正确
- [ ] 消息保存到数据库

#### 响应内容验证
- [ ] 初始请求返回 clarificaiton 类型
- [ ] 包含启发式提问
- [ ] 完成触发词返回 summary 类型
- [ ] metadata.stage 正确变化
