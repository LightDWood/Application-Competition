### 2.2 核心业务流程图

#### 流程 1：用户注册与登录流程

```mermaid
flowchart TD
    Start([用户访问网站]) --> CheckAuth{是否登录？}
    
    CheckAuth -->|未登录 | ShowLogin[显示登录页面]
    CheckAuth -->|已登录 | Redirect[跳转到首页]
    
    ShowLogin --> HasAccount{已有账号？}
    HasAccount -->|是 | InputLogin[输入用户名密码]
    HasAccount -->|否 | GoRegister[跳转到注册页面]
    
    GoRegister --> InputRegister[填写注册信息<br/>用户名/邮箱/密码]
    InputRegister --> ValidateRegister{信息验证}
    ValidateRegister -->|失败 | ShowRegError[显示错误提示]
    ValidateRegister -->|成功 | CreateUser[创建用户记录]
    
    CreateUser --> HashPassword[密码 bcrypt 加密]
    HashPassword --> SaveDB[(保存到数据库)]
    SaveDB --> GenerateJWT[生成 JWT Token]
    
    InputLogin --> ValidateLogin{验证凭据}
    ValidateLogin -->|失败 | ShowLoginError[显示错误提示]
    ValidateLogin -->|成功 | GenerateJWT
    
    GenerateJWT --> StoreToken[存储 Token 到 localStorage]
    StoreToken --> StoreUserInfo[存储用户信息到 localStorage]
    StoreUserInfo --> Redirect
    
    Redirect --> End([流程结束])
```

**详细步骤说明：**

| 步骤 | 组件 | 技术实现 | 说明 |
|------|------|----------|------|
| 1. 填写信息 | Frontend | Vue Form + Element Plus | 表单验证在前端完成 |
| 2. 提交请求 | Frontend | Axios POST /api/auth/register | 携带用户名/邮箱/密码 |
| 3. 后端验证 | Backend | Express Validator | 检查邮箱唯一性、密码强度 |
| 4. 密码加密 | Backend | bcryptjs | 10 轮 salt 加密 |
| 5. 保存数据库 | Backend | Sequelize User.create() | 写入 users 表 |
| 6. 生成 Token | Backend | jsonwebtoken.sign() | JWT 包含 userId 和 role |
| 7. 返回前端 | Backend | Response JSON | {code: 0, data: {user, token}} |
| 8. 本地存储 | Frontend | localStorage | 存储 token 和 userInfo |
| 9. 路由跳转 | Frontend | Vue Router.push() | 跳转到首页或目标页面 |

#### 流程 2：创意工坊需求澄清流程

```mermaid
flowchart TD
    Start([用户进入创意工坊]) --> CheckAuth{是否登录？}
    
    CheckAuth -->|未登录 | RedirectLogin[跳转到登录页]
    CheckAuth -->|已登录 | ShowChat[显示聊天窗口]
    
    ShowChat --> UserInput[用户输入需求描述]
    UserInput --> IntentRecognize[意图识别<br/>skillService.js]
    
    IntentRecognize --> IntentType{意图类型}
    
    IntentType -->|QA 问答 | HandleQA[QA 处理器<br/>直接回答问题]
    IntentType -->|REQUIREMENT 需求 | HandleReq[需求处理器<br/>启发式引导]
    IntentType -->|CHAT 闲聊 | HandleChat[闲聊处理器<br/>友好交流]
    
    HandleReq --> Analyze5W2H[5W2H 完整性分析]
    Analyze5W2H --> RiskAnalysis[风险预警检测]
    RiskAnalysis --> DependencyAnalysis[依赖关系发现]
    DependencyAnalysis --> GenerateResponse[生成引导性回复]
    
    HandleQA --> GenerateQA[生成简洁答案]
    HandleChat --> GenerateChat[生成友好回复]
    
    GenerateResponse --> StreamOutput[流式输出<br/>SSE]
    GenerateQA --> StreamOutput
    GenerateChat --> StreamOutput
    
    StreamOutput --> UserFeedback{用户反馈}
    UserFeedback -->|继续补充 | UserInput
    UserFeedback -->|需求已清晰 | GenerateDoc[生成需求文档]
    
    GenerateDoc --> SaveSession[保存会话到数据库]
    SaveSession --> SaveMessages[保存对话记录]
    SaveMessages --> SaveDocument[保存文档草稿]
    SaveDocument --> OptionalFeishu{同步飞书？}
    
    OptionalFeishu -->|是 | CreateFeishuDoc[创建飞书文档]
    OptionalFeishu -->|否 | SkipFeishu[跳过飞书]
    
    CreateFeishuDoc --> AddCollaborator[添加协作者]
    AddCollaborator --> UpdateLocalDoc[更新本地文档]
    UpdateLocalDoc --> End
    
    SkipFeishu --> End([流程结束])
```

### 4.2 核心 API 接口详解

#### 4.2.1 认证接口

**POST /api/auth/login**

```javascript
// 请求
{
  "username": "admin",
  "password": "admin123"
}

// 响应 (成功)
{
  "code": 0,
  "data": {
    "user": {
      "id": "uuid",
      "username": "admin",
      "email": "admin@ai-contest.com",
      "role": "admin"
    },
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
}

// 响应 (失败)
{
  "code": 401,
  "message": "用户名或密码错误"
}
```

**POST /api/auth/register**

```javascript
// 请求
{
  "username": "zhangsan",
  "email": "zhangsan@haier.com",
  "password": "securePassword123"
}

// 响应
{
  "code": 0,
  "data": {
    "user": { ... },
    "token": "..."
  }
```

## 六、安全架构

### 6.1 认证与授权

**JWT Token 流程：**

```mermaid
sequenceDiagram
    participant Client as 客户端
    participant Server as 后端
    participant DB as 数据库
    
    Client->>Server: POST /api/auth/login<br/>{username, password}
    Server->>DB: 查询用户
    DB-->>Server: 用户数据
    Server->>Server: bcrypt 验证密码
    Server->>Server: JWT.sign({id, role})
    Server-->>Client: {token, user}
    Client->>Client: localStorage 存储
    
    Note over Client,Server: 后续请求
    Client->>Server: GET /api/user/profile<br/>Authorization: Bearer {token}
    Server->>Server: JWT.verify(token)
    Server->>Server: 检查角色权限
    Server-->>Client: {user}
```

**权限控制矩阵：**

| 资源/操作 | 匿名 | 普通用户 | 管理员 |
|-----------|------|----------|--------|
| 访问首页 | ✅ | ✅ | ✅ |
| 查看培训 | ❌ | ✅ | ✅ |
| 管理培训 | ❌ | ❌ | ✅ |
| 使用创意工坊 | ❌ | ✅ | ✅ |
| 查看会话列表 | ❌ | ✅ (仅自己) | ✅ (全部) |
| 删除会话 | ❌ | ✅ (仅自己) | ✅ (全部) |
| 查看用户列表 | ❌ | ❌ | ✅ |
| 系统管理 | ❌ | ❌ | ✅ |

### 6.2 数据安全

## 七、部署架构

### 7.1 Nginx 配置

**生产环境部署：**

```nginx
upstream ai_contest_backend {
    server 127.0.0.1:3001;
    keepalive 64;
}

server {
    listen 80;
    server_name _;
    client_max_body_size 20M;

    # API 反向代理
    location /ai-contest/api {
        rewrite ^/ai-contest/api/(.*) /api/$1 break;
        proxy_pass http://ai_contest_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 文件上传
    location /ai-contest/uploads {
        rewrite ^/ai-contest/uploads/(.*) /uploads/$1 break;
        proxy_pass http://ai_contest_backend;
    }

    # 静态文件
    location /ai-contest {
        alias /var/www/ai-contest/frontend;
        index index.html;
        try_files $uri $uri/ /ai-contest/index.html;
    }
}
```

### 8.1 前端优化
