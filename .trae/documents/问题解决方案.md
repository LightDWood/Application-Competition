## 问题1解决方案 (AI服务不可用) - 已修复

### 根本原因分析

通过调试发现：

1. **直接调用API测试**: ✅ 成功
   ```
   curl -X POST https://modelapi-test.haier.net/model/v1/chat/completions
   Status: 200
   Response: {"id":"chatcmpl-...","choices":[{"message":{"content":"..."}}}]
   ```

2. **通过Chat接口调用**: ❌ 失败
   ```
   API request failed: 400 - {"error":{"message":"2 validation errors for ValidatorIterator\n
   0.ChatCompletionContentPartTextParam\n
   Input should be a valid dictionary..."}}
   ```

3. **原因定位**:
   - 问题出在消息历史格式转换时
   - 历史消息中的content可能是 `{type: 'text', text: '...'}`
   - 代码 `m.content?.text || m.content` 在某些边界情况下返回了非字符串值

### 修复方案

**修改文件**: `server/routes/v2.js` 第546-560行

**修改前**:
```javascript
const modelRequest = {
  messages: [
    { role: 'system', content: compiledPrompt.system + '\n\n' + compiledPrompt.agent },
    ...history.map(m => ({
      role: m.role,
      content: m.content?.text || m.content
    })),
    { role: 'user', content }
  ],
```

**修改后**:
```javascript
const modelRequest = {
  messages: [
    { role: 'system', content: compiledPrompt.system + '\n\n' + compiledPrompt.agent },
    ...history.map(m => {
      const content = typeof m.content === 'string' ? m.content : (m.content?.text || '')
      return { role: m.role, content }
    }),
    { role: 'user', content }
  ],
```

**验证结果**:

## 问题2解决方案 (会话列表缺少标题) - 已修复

### 根本原因分析

会话从磁盘加载时，title字段可能为空，需要从metadata.title或第一条消息中提取。

### 修复方案

在 `SessionService.js` 中添加会话标题提取逻辑：

```javascript
extractSessionFromMessages(sessionId, agentId, messageLines) {
  // 从第一条用户消息中提取标题
  let firstUserMessage = null;
  for (const line of messageLines) {
    try {
      const msg = JSON.parse(line);
      if (msg.role === 'user' && !firstUserMessage) {
        const content = typeof msg.content === 'string' ? msg.content : (msg.content?.text || '');
        firstUserMessage = content.substring(0, 50);
      }
    } catch (e) { continue; }
  }

  return {
    title: firstUserMessage || '历史会话',
    metadata: { title: firstUserMessage || '历史会话' },
    // ...
  };
}
```
