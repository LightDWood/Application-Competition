# 需求收敛归档功能修复计划

## 问题清单

### 严重问题
1. **归档未实际执行** - 用户输入"归档"后，AI只返回摘要文本，`ArtifactService.saveArtifact()` 未被调用

### 中等问题
2. **Spec文档结构不完整** - 对话中生成的Spec缺少多个必要章节
3. **5维度收集顺序有误** - 第1轮问了"成功标准"而非"做什么"

### 轻微问题
4. **归档摘要过于简略** - 应该保存完整文档而不是摘要

---

## 修复方案

### Step 1: 完成归档逻辑集成

**位置**: `ai-contest-web/server/routes/v2.js` 的 `post('/sessions/:id/chat')` endpoint

**修改内容**:
1. 在处理用户消息时，检测是否包含"归档"关键字
2. 如果检测到归档：
   - 获取会话历史
   - 调用 `generateRequirementDoc()` 生成完整文档
   - 调用 `artifactService.saveArtifact()` 保存
   - 返回下载链接而非发送给LLM

**代码逻辑**:
```javascript
// 在 router.post('/sessions/:id/chat') 中
if (content.includes('归档')) {
  // 1. 获取会话历史
  const history = sessionService.getSessionHistory(id)

  // 2. 生成完整文档
  const specTemplate = loadSpecTemplate()
  const docContent = generateRequirementDoc(id, history, specTemplate)

  // 3. 保存到 ArtifactService
  await artifactService.saveArtifact(id, {
    type: 'requirement-doc',
    name: `需求文档_${Date.now()}`,
    content: docContent
  })

  // 4. 返回确认消息和下载链接
  res.write(`data: ${JSON.stringify({ type: 'chunk', content: '✅ 归档成功！\n' })}\n\n`)
  res.write(`data: ${JSON.stringify({ type: 'chunk', content: '📄 文档已保存到 artifacts/${id}/ 目录\n' })}\n\n`)
  res.write(`data: ${JSON.stringify({ type: 'complete', metadata: { archived: true } })}\n\n`)
  res.end()
  return
}
```
