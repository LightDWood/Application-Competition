## 四、发现的问题与修复

### 4.1 发现的问题

**问题描述**: 注册接口返回 500 错误

**错误信息**:
```
Error: Invalid value { username: 'testuser001' }
at Object.escape (sequelize\lib\sql-string.js:54:11)
```

**根本原因**: 
在 [`user.js`](file://d:\DFX\Code\AIbisai-wangzhan\ai-contest-web\server\routes\user.js) 文件中使用了 MongoDB 的查询语法 (`$or`),但项目使用的是 MySQL 数据库。

**问题代码** (第 20-27 行):
```javascript
const existingUser = await User.findOne({
  where: {
    $or: [  // ❌ MongoDB 语法，不适用于 Sequelize + MySQL
      { username },
      { email }
    ]
  }
})
```

### 4.2 修复方案

**修复步骤**:
1. 从 Sequelize 导入 `Op` 对象
2. 将 `$or` 替换为 `[Op.or]`

**修复后的代码**:
```javascript
import { Op } from 'sequelize'  // ✅ 添加导入

const existingUser = await User.findOne({
  where: {
    [Op.or]: [  // ✅ Sequelize 正确语法
      { username },
      { email }
    ]
  }
})
```

**修复范围**:
- 注册接口 (第 20-27 行)
- 登录接口 (第 93-100 行)

### 4.3 修复验证

修复后重新执行测试，所有测试用例均通过。
