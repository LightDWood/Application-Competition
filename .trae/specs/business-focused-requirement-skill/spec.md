# 业务聚焦式需求收敛 Skill 优化 Spec

## Why

当前需求收敛存在以下问题：

1. 机械提问 - 不考虑用户上下文，逐条硬编码提问
2. 技术冗余 - 用户被迫参与技术设计讨论，增加认知负担
3. 输出混乱 - 会话输出与归档文档无区分，用户体验差
4. 来源不明 - 回复缺少来源标注，无法追溯

需要实现：**业务聚焦 + 动态智能收敛 + 分层输出** 的新一代需求收敛能力。

## What Changes

* **动态智能收敛**：基于用户描述动态生成问题，识别模糊点智能追问

* **业务聚焦对话**：用户只需讨论业务维度（做什么/给谁用/约束/成功标准/痛点）

* **技术自动生成**：AI 基于业务需求自动推荐技术方案，用户只需确认

* **分层输出**：会话精简（≤200字/轮）+ 完整文档归档

* **来源标注**：每条回复标注 Agent/Skill/Mode

* **Spec 模板引用**：规范文档模板存储于 resources/Spec.md

## Impact

* **Affected files**:

  * `skills/requirement-convergence/SKILL.md` - 重写核心 SOP

  * `skills/requirement-convergence/resources/Spec.md` - 新增模板

  * `config/agents/default-agent/soul.yaml` - 更新身份定义

  * `config/system.yaml` - 添加输出规范

  * `server/services/core/ArtifactService.js` - 支持文档归档

## ADDED Requirements

### Requirement: 动态智能收敛

The system SHALL provide 动态智能收敛能力，包括：

#### 上下文感知分析

* **WHEN** 用户输入需求描述时

* **THEN** AI 实时分析并提取：

  * 已明确的实体（项目类型、用户角色、行业领域）

  * 模糊描述（效率低、系统慢 - 追问具体化）

  * 缺失要素（按 5 维度检查完整性）

#### 智能追问策略

* **WHEN** 检测到模糊或缺失信息时

* **THEN** AI 选择追问策略：

  * 模糊追问：「你说的"效率低"具体指什么场景？」

  * 补全追问：「目前日均处理量是多少？」

  * 确认追问：「你的意思是...对吗？」

#### 渐进式收敛

* **WHEN** 收集到足够信息时

* **THEN** AI 主动汇总并进入下一阶段

* 不强制固定顺序，按需动态调整

***

### Requirement: 业务聚焦对话

The system SHALL 只与用户讨论业务维度，不涉及技术细节：

#### 用户只需决定的5个维度

| 维度   | 用户问题示例   | 说明       |
| ---- | -------- | -------- |
| 做什么  | 想解决什么问题？ | 核心目标     |
| 给谁用  | 目标用户是谁？  | 角色       |
| 约束条件 | 有哪些限制？   | 预算/时间/资源 |
| 成功标准 | 怎么算成功？   | 量化指标     |
| 当前痛点 | 现在最大问题？  | 现状       |

#### AI 自动生成的技术维度

| 维度   | 用户是否参与 | 方式      |
| ---- | ------ | ------- |
| 技术选型 | ❌ 否    | AI 自动推荐 |
| 架构设计 | ❌ 否    | AI 自动生成 |
| 实施路线 | ❌ 否    | AI 自动推算 |
| 风险评估 | ❌ 否    | AI 自动分析 |

***

### Requirement: 分层输出机制

The system SHALL 提供分层输出机制：

#### 会话层（精简）

* **WHEN** 对话过程中

* **THEN** 每轮输出 ≤200 字

* **AND** 使用表格/选项式交互

* **AND** 显示进度指示器

#### 归档层（完整）

* **WHEN** 用户确认生成文档时

* **THEN** AI 生成完整规范文档（无字数限制）

* **AND** 自动保存到制品服务

* **AND** 提供下载链接

***

### Requirement: 来源标注

The system SHALL 在每条回复末尾标注来源：

#### 标注格式

```
---
🤖 Agent: {agentName}
📋 Skill: {skillName} v{version}
🎯 Mode: {currentMode}
📊 完整度: {completeness}%
```

#### Mode 枚举

| Mode    | 说明      |
| ------- | ------- |
| 🎯 智能收敛 | 业务维度收集  |
| 🔍 澄清追问 | 模糊/缺失追问 |
| 💡 智能推荐 | 基于上下文推荐 |
| ✅ 阶段确认  | 信息汇总确认  |
| 🔧 技术推荐 | AI 技术方案 |
| 📄 制品生成 | 文档生成中   |

***

### Requirement: Spec 模板引用

The system SHALL 提供规范文档模板：

#### 模板位置

```
skills/requirement-convergence/resources/Spec.md
```

#### 模板结构

```
1. 基本信息（用户填写）
2. 设计目标（用户填写）
3. 业务现状（用户填写）
4. 目标维度分析（用户填写）
5. 方案设计
   - 5.1-5.3 用户参与讨论
   - 5.4+ AI 自动生成
6. 非功能性需求（AI 自动推荐）
7. 接口设计（AI 自动生成）
8. 数据需求（AI 自动生成）
9. 依赖分析（AI 自动生成）
10. 附录（AI 自动生成）
```

***

### Requirement: 制品归档服务

The system SHALL 支持需求文档归档：

#### 归档触发

* **WHEN** 用户确认需求后

* **THEN** AI 生成完整 Markdown 文档

* **AND** 保存到 ArtifactService

#### 下载格式

* Markdown（.md）- 主要格式

* 后续可扩展 PDF

## 核心对话示例

### 场景：用户模糊描述

**用户**：「我想做一个提升效率的系统」

**AI 分析**：

```
⚠️ 模糊点：
- 什么场景的效率低？
- 现有流程是怎样的？

✅ 自动推断：
- 项目类型 = 效率提升类
- 目标用户 = 待确认
```

**AI 回复**：

```
💡 明白了！你想做效率提升类项目。

📊 **需求进度**（5项核心，已完成 1/5）

| # | 维度 | 状态 | 内容 |
|---|------|------|------|
| 1 | 做什么 | ✅ | 效率提升 |
| 2 | 给谁用 | ⏳ | 待确认 |
| 3 | 约束 | ⏳ | 待确认 |
| 4 | 成功标准 | ⏳ | 待确认 |
| 5 | 痛点 | ⏳ | 待确认 |

**第2问：给谁用？**
🌟 [A] 内部员工
[B] 外部客户
[C] 两者都有

💬 回复选项字母，如：A

---
🤖 Agent: AI助手
📋 Skill: 需求收敛 v3.0
🎯 Mode: 智能收敛
📊 完整度: 20%
```

***
