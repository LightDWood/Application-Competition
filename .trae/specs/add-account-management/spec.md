# 后台账号管理功能规范

## Why

当前系统已有用户注册和登录功能，但缺乏后台管理界面来查看和管理用户账号。需要添加账号管理功能，支持管理员查看用户信息、角色管理（普通用户/管理员），并提供跳转到用户文档管理页查看用户数据的能力。

## What Changes

* ✅ **新增用户角色系统**：一般用户/管理员两种角色

* ✅ **新增账号管理 Tab**：在后台管理页面添加"账号管理"标签页

* ✅ **角色权限控制**：管理员可在菜单栏看到后台管理入口，一般用户看不到

* ✅ **用户信息查看**：支持查看用户注册信息（用户名、邮箱、注册时间等）

* ✅ **文档管理跳转**：支持单点跳转到用户的文档管理页查看用户数据

## Impact

* **Affected specs**: 无

* **Affected code**:

  * `src/views/TrainingAdmin.vue` - 后台管理页面（添加账号管理 Tab）

  * `src/components/Header.vue` - 顶部导航栏（添加角色判断逻辑）

  * `src/views/user/Login.vue` - 登录页面（添加角色判断和跳转）

  * `server/models/User.js` - 用户模型（添加角色字段）

  * `server/routes/user.js` - 用户路由（添加角色相关接口）

  * `src/api/user.js` - 用户 API（添加角色相关方法）

## ADDED Requirements

### Requirement: 用户角色系统

The system SHALL provide 用户角色分类能力：

#### 角色定义

* **一般用户（User）**：

  * 可以访问创意工坊（需求收敛系统）

  * 可以查看和管理自己的会话和文档

  * 无法访问后台管理页面

* **管理员（Admin）**：

  * 拥有一般用户的所有权限

  * 可以访问后台管理页面

  * 可以管理培训资料、报名信息、账号等

  * 可以看到后台管理菜单入口

#### 角色存储

* **WHEN** 用户注册或创建时

* **THEN** 系统自动分配角色：

  * 默认角色：一般用户（User）

  * 管理员角色：Admin

  * 角色字段存储在用户数据库中

***

### Requirement: 账号管理 Tab

The system SHALL provide 账号管理界面：

#### 界面布局

* **位置**：后台管理页面的第四个 Tab（位于"网站二维码"之后）

* **Tab 名称**：账号管理

* **内容区域**：显示用户列表

#### 用户列表功能

* **WHEN** 用户进入账号管理 Tab

* **THEN** 显示以下信息：

  * 用户 ID（序号）

  * 用户名

  * 邮箱

  * 角色（一般用户/管理员）

  * 注册时间

  * 操作按钮：查看文档、删除用户

#### 用户列表工具栏

* **左侧**：显示统计信息（共 X 条用户记录）

* **右侧**：搜索框（按用户名/邮箱搜索）、筛选器（按角色筛选）

***

### Requirement: 查看用户文档

The system SHALL provide 跳转到用户文档管理页的能力：

#### 跳转逻辑

* **WHEN** 管理员点击"查看文档"按钮

* **THEN** 系统：

  * 在新标签页打开该用户的文档管理页

  * URL 格式：`/requirement/documents?userId={userId}`

  * 显示该用户的所有文档列表

#### 权限控制

* **WHEN** 非管理员尝试访问其他用户的文档

* **THEN** 系统拒绝访问并显示错误提示

***

### Requirement: 菜单权限控制

The system SHALL provide 基于角色的菜单显示控制：

#### 后台管理菜单显示逻辑

* **WHEN** 用户登录成功后

* **THEN** 系统检查用户角色：

  * 如果是管理员：在顶部导航栏显示"后台管理"菜单

  * 如果是一般用户：不显示"后台管理"菜单

#### 路由守卫增强

* **WHEN** 用户尝试访问后台管理页面

* **THEN** 系统验证：

  * 检查用户是否已登录（admin\_token）

  * 检查用户是否为管理员角色

  * 如果任一检查失败，跳转到登录页

***

### Requirement: 用户管理 API

The system SHALL provide 用户管理相关 API：

#### 获取用户列表

* **Endpoint**: `GET /api/admin/users`

* **参数**：

  * `page`（页码，默认 1）

  * `pageSize`（每页数量，默认 10）

  * `search`（搜索关键词，可选）

  * `role`（角色筛选，可选）

* **返回**：用户列表（分页）

#### 获取用户详情

* **Endpoint**: `GET /api/admin/users/:id`

* **返回**：用户详细信息

#### 删除用户

* **Endpoint**: `DELETE /api/admin/users/:id`

* **权限**：仅管理员可操作

* **返回**：删除结果

#### 更新用户角色

* **Endpoint**: `PUT /api/admin/users/:id/role`

* **参数**：`role`（'user' 或 'admin'）

* **权限**：仅管理员可操作

* **返回**：更新结果

***

## MODIFIED Requirements

### Requirement: 登录跳转逻辑（原登录功能）

**修改前**：登录成功后统一跳转到首页\
**修改后**：

* 管理员登录：跳转到后台管理页面（`/training-admin`）

* 一般用户登录：跳转到创意工坊（`/requirement`）或携带的 redirect 参数

### Requirement: 导航栏菜单（原 Header 组件）

**修改前**：固定显示所有菜单项\
**修改后**：

* "后台管理"菜单项仅对管理员显示

* 根据用户角色动态渲染菜单

***

无

### User 模型增强

```javascript
{
  id: UUID,
  username: String,
  email: String,
  password: String (hashed),
  role: String,  // 新增：'user' | 'admin'
  createdAt: Date,
  updatedAt: Date
}
```

* **用户名**: admin

* **密码**: admin123

* **角色**: admin

* **用途**: 系统初始化时创建
