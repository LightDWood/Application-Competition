# Requirement Convergence Skill（需求收敛技能 v4.0）

## 核心理念

- 🎯 **赋能大模型**：提供框架和信息，让AI自主发挥引导能力
- 🔄 **动态引导**：基于上下文智能调整问题，而非静态选项
- 📝 **结构化记录**：完整记录会话过程，为制品生成提供完整上下文
- ⚡ **高效输出**：会话精简（≤200字/轮），完整文档归档

### 三大核心原则

| 原则 | 说明 |
|------|------|
| 动态引导 | 问题根据用户上下文实时生成，而非预设的固定选项 |
| 充分上下文 | 提供LLM所需的所有信息（进度、用户背景、对话历史） |
| 结构化记录 | 每一轮交互都有完整记录，支持后续追溯和制品生成 |

---

## 触发条件

### ✅ 应该触发
- 用户说"我想做"/"需要开发"/"规划一个"
- 用户需求模糊不清
- 用户说"帮我理一下需求"
- 用户讨论功能但没想清楚细节

### ❌ 不应触发
- 用户明确要求写代码/实现功能
- 用户已有清晰的需求文档
- 用户只是闲聊

## 意图识别引擎（Intent Recognition Engine）

### 实体提取规则

| 实体类型 | 识别规则 | 示例 |
|----------|----------|------|
| 数字 | 阿拉伯数字、汉字数字、百分比 | "3个月"、"30%"、"一千" |
| 时间 | 日期、时长、截止日期 | "下周"、"两周内"、"3月前" |
| 用户类型 | 内部/外部、特定角色 | "员工"、"客户"、"财务" |
| 技术术语 | 特定技术名称 | "微信小程序"、"React" |
| 场景术语 | 业务流程场景 | "审批"、"报销"、"订单" |

### 意图分类

| 意图类型 | 识别特征 | AI响应策略 |
|----------|----------|------------|
| 明确选择 | 选项字母、完整选项描述 | 记录选择，继续下一维度 |
| 模糊描述 | 未直接匹配选项的描述性语言 | 提取实体，追问澄清 |
| 跳过 | "不确定"、"跳过"、"待定" | 记录低置信度，继续下一维度 |
| 纠正 | "不是"、"不对"、"其实" | 回退上一轮，重新收集 |
| 确认 | "是的"、"对的"、"正确" | 提升置信度，标记已确认 |

### 维度匹配矩阵

| 用户输入维度 | 匹配的核心维度 | 映射规则 |
|--------------|----------------|----------|
| 效率、成本、增长 | goal | 直接映射 |
| 员工、客户、用户群 | targetUsers | 直接映射 |
| 时间、预算、技术栈 | constraints | 直接映射 |
| 指标、目标、上线 | successCriteria | 直接映射 |
| 问题、痛点、不满意 | painPoints | 直接映射 |
| 其他描述 | 由LLM推断 | 基于语义映射到最近维度 |

## 置信度计算规则（Confidence Calculation）

### 5维度置信度评估

| 维度 | 高置信度(≥0.9) | 中置信度(0.7-0.9) | 低置信度(<0.7) |
|------|----------------|-------------------|----------------|
| 做什么 | 明确描述+多证据支撑 | 明确描述但证据单一 | 模糊描述或笼统 |
| 给谁用 | 具体用户群+使用场景 | 笼统分类（如"用户"） | 不确定或未说明 |
| 约束条件 | 具体数字+明确截止 | 定性描述但无量化 | 无约束或"都可以" |
| 成功标准 | 可量化指标+目标值 | 定性成功描述 | 不确定或"做好就行" |
| 当前痛点 | 具体场景+问题描述 | 笼统问题描述 | 不确定或未提及 |

### 置信度计算公式

```
dimension_confidence = base_confidence * evidence_multiplier * clarity_multiplier

其中：
- base_confidence: 基础置信度（0.5）
- evidence_multiplier: 证据倍数（+0.1/每个独立证据，上限+0.3）
- clarity_multiplier: 清晰度倍数（明确描述=1.3，模糊描述=1.0）
```

## 完整度评估（Completeness Evaluation）

### 计算公式

```
completeness = Σ(dimension_confidence × dimension_weight) / Σ(dimension_weight)

维度权重：
- goal: 0.25
- targetUsers: 0.20
- constraints: 0.15
- successCriteria: 0.20
- painPoints: 0.20
```
