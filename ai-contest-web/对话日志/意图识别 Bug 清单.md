## 📋 Bug 详细描述

### BUG-IR-001: 问答意图识别不全

**问题描述**:  
`intentRecognizer.js` 中的问答意图模式不完整，导致常见问题被误判为闲聊。

**影响**:
- 用户问 "你有什么能力" 被误判为 CHAT（闲聊），而不是 QA（问答）
- 用户问 "你能做什么" 被误判为 CHAT
- 用户问 "介绍一下你自己" 被误判为 CHAT
- 所有询问 AI 能力、功能的问题都无法得到正确回答

**根本原因**:
```javascript
// 当前的 QA 模式只匹配特定格式
QA: [
  /什么 (模型 | 技术 | 功能|AI|系统 | 工具)/i,  // 必须包含这些词
  /为什么/i,
  /如何 | 怎么/i,
  // ... 缺少 "能力"、"能做什么" 等模式
]
```

**不符合预期的案例**:
```
❌ "你有什么能力" → 被误判为 CHAT
❌ "你能做什么" → 被误判为 CHAT  
❌ "介绍一下你自己" → 被误判为 CHAT
❌ "你会什么" → 被误判为 CHAT
❌ "你的功能是什么" → 被误判为 CHAT
```

**预期行为**:
```
✅ "你有什么能力" → 应该识别为 QA
✅ "你能做什么" → 应该识别为 QA
✅ "介绍一下你自己" → 应该识别为 QA
```

### BUG-IR-002: 需求意图识别不全

**问题描述**:  
`intentRecognizer.js` 中的需求分析意图模式不完整，导致常见需求表达被误判为闲聊。

**影响**:
- 用户说 "我想做一个咨询订阅的工具" 被误判为 CHAT（闲聊），而不是 REQUIREMENT（需求分析）
- 用户说 "我需要一个 XX 系统" 被误判为 CHAT
- 所有包含 "咨询"、"订阅" 等行业术语的需求无法被正确识别

**根本原因**:
```javascript
// 当前的 REQUIREMENT 模式只匹配特定格式
REQUIREMENT: [
  /我要做/,
  /我想 (做 | 实现 | 开发 | 创建)/i,
  /需要 (一个 | 个|套|款)/i,
  // ... 缺少 "咨询"、"订阅" 等行业场景
]
```

**不符合预期的案例**:
```
❌ "我想做一个咨询订阅的工具" → 被误判为 CHAT
❌ "我需要一个合同管理系统" → 被误判为 CHAT
❌ "想要一个电商小程序" → 被误判为 CHAT
❌ "帮我开发一个 CRM 系统" → 被误判为 CHAT
```

**预期行为**:
```
✅ "我想做一个咨询订阅的工具" → 应该识别为 REQUIREMENT
✅ "我需要一个合同管理系统" → 应该识别为 REQUIREMENT
✅ "想要一个电商小程序" → 应该识别为 REQUIREMENT
```

### BUG-IR-003: 意图识别模式过于僵化

**问题描述**:  
当前意图识别完全依赖正则表达式匹配，缺乏语义理解能力。

**影响**:
- 用户表达方式稍有变化就无法识别
- 不支持同义词、近义词识别
- 不支持上下文理解

**不符合预期的案例**:
```
❌ "帮我弄个网站" → "弄" 不在模式列表中
❌ "搞个 APP" → "搞" 不在模式列表中
❌ "整个系统" → "整" 不在模式列表中
```

**建议优化方向**:
1. 扩展口语化表达模式
2. 添加同义词映射
3. 引入简单的语义分析

### 优化 BUG-IR-003: 增强意图识别（可选）

```javascript
// 添加同义词映射
const SYNONYMS = {
  '做': ['弄', '搞', '整', '开发', '实现', '创建'],
  '需要': ['想要', '希望有', '需要一个'],
  '能力': ['功能', '本事', '技能', '会什么', '能做什么']
};

// 在匹配前先进行同义词替换
function normalizeInput(input) {
  let normalized = input;
  for (const [original, synonyms] of Object.entries(SYNONYMS)) {
    for (const synonym of synonyms) {
      normalized = normalized.replace(new RegExp(synonym, 'gi'), original);
    }
  }
  return normalized;
}
```
