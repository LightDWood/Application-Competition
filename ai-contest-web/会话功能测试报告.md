## Bug 清单

### Bug #1: 会话服务内存状态未持久化

**严重程度**: P1 (高)

**Bug描述**:
SessionService 使用内存Map存储会话数据，但不会从磁盘加载历史会话。当服务重启后，所有会话列表为空，用户无法查看历史会话。

**影响范围**:
- 会话列表页面显示空
- 无法加载历史会话详情
- 所有基于会话ID的操作都返回 404

**根本原因**:
```javascript
// SessionService.js
class SessionService {
  constructor() {
    this.sessions = new Map();  // 仅内存存储
    this.messages = new Map();
  }
  // ...
  getUserSessions(userId) {
    const userSessions = [];
    for (const session of this.sessions.values()) {  // 只遍历内存
      if (session.userId === userId) {
        userSessions.push(session);
      }
    }
    return userSessions;
  }
}
```

**复现步骤**:
1. 重启服务器 (node server/app.js)
2. 访问 http://localhost:5174/ai-contest/requirement/sessions
3. 观察控制台输出

**预期行为**: 应显示所有历史会话

**实际行为**: 显示空列表

**修复建议**:
在 SessionService 初始化时扫描 `agents/*/sessions/*.jsonl` 文件，加载历史会话到内存Map。

### Bug #2: 归档生成的文档未在前端文档列表显示

**严重程度**: P2 (中)

**Bug描述**:
归档功能在后端能正常生成文档并保存到 `server/artifacts/{sessionId}/` 目录，但前端文档管理页面 (`DocumentList.vue`) 的下载功能是空实现。

**影响范围**:
- 用户无法在前端看到归档生成的文档列表
- 无法通过前端界面下载需求文档

**根本原因**:
```javascript
// DocumentList.vue
const downloadDocument = (doc) => {
  console.log('Download document:', doc)
  // 空实现，未实际下载
}
```

**修复建议**:
1. 添加文档列表API (`GET /api/v2/documents`)
2. 实现文档下载功能
3. 关联归档生成的artifact
